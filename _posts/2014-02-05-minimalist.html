Minimalist HTML5

HTML from its inception in 1991 has been about semantically marking up documents. With the growth of graphic displays, web developers have been trying to transform HTML into a style sheet language, but the <font> tag and imprecise attributes left much to be desired, and so CSS was created in 1996. The ideal of separating presentation and content also brought forth XML, an HTML-like markup language distilled into pure semantics and zero presentation/styling demands, leaving presentation to separate style sheets in CSS or XSL. Sadly, though, XML's requirement to be well-formed proved too strict as kludging HTML into attractive web pages had become standard practice.

Today the same content may be picked up and displayed in various different contexts. You main content is now scraped to display in Google searches, Facebook posts, aggragate sites, maybe even previews on your own homepage.

And now, with the myriad of different devices that connect to the Internet and display web content, it is necessary to show the same content with many different layouts.

Use minimalist, semantic HTML5 with no superfluous, unnecessary markup.

	float: none; clear: both;

	display: block; content: ''; float: none; clear: both;
